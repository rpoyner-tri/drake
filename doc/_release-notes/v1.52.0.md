# Breaking changes since v1.51.1

<!-- <relnotes for breaking-changes go here> -->

* [breaking change fix] TBD [common,tools,geometry,multibody] [common] Move spdlog to an implementation detail ([#24202][_#24202])  # The common/text_logging.h header no longer includes spdlog. Instead, we provide a new class drake::logger with a subset of the same API. This narrows our third-party include footprint. It also opens the door to turning off spdlog (with its complicated hierarchy and sinks) while still keeping simple stderr console logging enabled. The DRAKE_LOGGER_DEBUG and DRAKE_LOGGER_TRACE macros are no longer conditioned on spdlog at compile time; they are only en/disabled based on NDEBUG. (Previously they were no-ops if spdlog was disabled or NDEBUG was set. Now, they are only no-op when NDEBUG is set.) Instead, the runtime check for should_log() will return false without spdlog. For the implementation, instead of having `#if HAVE_SPDLOG` guards inside a single text_logging.cc file, now we have separate files: - text_logging.cc for unconditional code - text_logging_impl_spdlog.cc for code with HAVE_SPDLOG - text_logging_impl_none.cc for code without HAVE_SPDLOG

# Notes


This release provides [pre-compiled binaries](https://github.com/RobotLocomotion/drake/releases/tag/v1.52.0) named
``drake-1.52.0-{noble|mac-arm64}.tar.gz``. See [Stable Releases](/from_binary.html#stable-releases) for instructions on how to use them.

Drake binary releases incorporate a pre-compiled version of [SNOPT](https://ccom.ucsd.edu/~optimizers/solvers/snopt/) as part of the
[Mathematical Program toolbox](https://drake.mit.edu/doxygen_cxx/group__solvers.html). Thanks to
Philip E. Gill and Elizabeth Wong for their kind support.
